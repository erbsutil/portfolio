keyDecisions:
  - decision: "Astro over Next.js"
    reasoning: "Next.js ships a React runtime to every page. For a content-heavy site, this overhead is unnecessary. Astro's zero-JS-by-default behavior fits the 'Performance as a Feature' requirement perfectly."
  - decision: "Islands Architecture for Interactivity"
    reasoning: "The hero carousel requires state management. Using a React island allows rich interactivity without taxing the rest of the page's initial load time."
  - decision: "End-to-End Testing Strategy"
    reasoning: "Unit testing React components inside an Astro environment introduces unnecessary complexity (ESM vs CommonJS errors). Playwright automates real-browser interactions to verify the compiled output, catching accessibility traps and broken critical user flows with high fidelity."
  - decision: "AI-Assisted Workflow"
    reasoning: "Using advanced AI coding agents allowed for rapid prototyping of boilerplate and layout patterns. My role shifted to technical direction, architectural constraints, and performance auditing."
  - decision: "Vanilla CSS over Tailwind"
    reasoning: "Astro's scoped styles provide modularity without abstraction overhead. CSS variables and local scoping offered more precise control with zero runtime cost."
